Module 1: HTML Basics

Learn the fundamental building blocks of HTML, including tags, elements, and document structure.

What is HTML?

HTML (HyperText Markup Language) is the standard language used to create and structure content on the web. It forms the backbone of every website, defining elements like text, images, links, and layout to make content visible and organized in browsers.

HTML elements are the building blocks of HTML pages and are represented by tags.

Provides the basic framework for all web pages.
Uses opening and closing tags to define elements.
Allows adding hyperlinks, images, audio, and video.
Attributes give extra information about elements like src, href, or alt.
HTTP is the mechanism by which HTML documents are transported across the internet.

Basic HTML Document Structure

Every HTML document follows a basic structure:

<!DOCTYPE html>
<html>
<head>
  <title>Page Title</title>
</head>
<body>
  <h1>My First Heading</h1>
  <p>My first paragraph.</p>
</body>
</html>

Let's break down each part:

  • <!DOCTYPE html>: Declares the document type and version of HTML
  • <html>: The root element of an HTML page
  • <head>: Contains meta information about the document
  • <title>: Specifies a title for the document
  • <body>: Contains the visible page content

HTML Elements and Tags

HTML elements are defined by a start tag, some content, and an end tag:

<tagname>Content goes here...</tagname>

Examples of HTML elements:

<h1>This is a heading</h1>
<p>This is a paragraph.</p>
<a href="https://www.example.com">This is a link</a>

Some important points about HTML elements:

  • HTML tags are not case sensitive, but lowercase is recommended
  • Some elements are empty (have no content and no closing tag)
  • Elements can be nested (contain other elements)

HTML Attributes

Attributes provide additional information about HTML elements. They are always specified in the start tag and usually come in name/value pairs like name="value".

<a href="https://www.example.com">Visit Example.com</a>

In this example, href is an attribute that specifies the URL the link goes to.

Common attributes include:

  • id: Specifies a unique id for an element
  • class: Specifies one or more class names for an element
  • style: Specifies an inline CSS style for an element
  • src: Specifies the source for embedded content
  • alt: Specifies alternative text for images

Module 2: Text Formatting

Master text elements, headings, paragraphs, lists, and semantic text markup.

Headings

HTML provides six levels of headings, from <h1> to <h6>. <h1> defines the most important heading, while <h6> defines the least important.

<h1>This is heading 1</h1>
<h2>This is heading 2</h2>
<h3>This is heading 3</h3>
<h4>This is heading 4</h4>
<h5>This is heading 5</h5>
<h6>This is heading 6</h6>

Important notes about headings:

  • Search engines use headings to index the structure and content of your web pages
  • Use headings to show the document structure (not to make text big or bold)
  • Each page should have one <h1> element for the main heading

Paragraphs and Line Breaks

The <p> element defines a paragraph. Browsers automatically add some white space (a margin) before and after each paragraph.

<p>This is a paragraph.</p>
<p>This is another paragraph.</p>

The <br> tag inserts a single line break. It is an empty tag, which means it has no end tag.

<p>This is<br>a paragraph<br>with line breaks.</p>

Text Formatting Elements

HTML provides various elements for formatting text with special meaning:

<b> - Bold text
<strong> - Important text
<i> - Italic text
<em> - Emphasized text
<mark> - Marked text
<small> - Smaller text
<del> - Deleted text
<ins> - Inserted text
<sub> - Subscript text
<sup> - Superscript text

Note: <b> and <i> define bold and italic text without any extra importance, while <strong> and <em> indicate that the content has strong importance or emphasis.

Lists

HTML provides three types of lists:

Unordered Lists

An unordered list starts with the <ul> tag. Each list item starts with the <li> tag.

<ul>
  <li>Coffee</li>
  <li>Tea</li>
  <li>Milk</li>
</ul>

Ordered Lists

An ordered list starts with the <ol> tag. Each list item starts with the <li> tag.

<ol>
  <li>Coffee</li>
  <li>Tea</li>
  <li>Milk</li>
</ol>

Description Lists

A description list is a list of terms, with a description of each term.

<dl>
  <dt>Coffee</dt>
  <dd>- black hot drink</dd>
  <dt>Milk</dt>
  <dd>- white cold drink</dd>
</dl>

Module 3: Links and Navigation

Create hyperlinks, navigation menus, and understand URL structures.

HTML Links - Hyperlinks

HTML links are defined with the <a> tag. The link's destination is specified in the href attribute.

<a href="https://www.example.com">Visit Example.com</a>

By default, links will appear as follows in all browsers:

  • An unvisited link is underlined and blue
  • A visited link is underlined and purple
  • An active link is underlined and red

You can change the link colors using CSS.

The target Attribute

By default, the linked page will be displayed in the current browser window. To change this, you can use the target attribute.

<a href="https://www.example.com" target="_blank">Visit Example.com in a new tab</a>
<a href="https://www.example.com" target="_self">Visit Example.com in the same tab (default)</a>
<a href="https://www.example.com" target="_parent">Visit Example.com in the parent frame</a>
<a href="https://www.example.com" target="_top">Visit Example.com in the full body of the window</a>

target="_blank" is commonly used to open links in a new tab or window.

Absolute vs. Relative URLs

URLs can be either absolute or relative:

Absolute URLs

An absolute URL contains the full address of the resource, including the protocol (http/https) and domain name.

<a href="https://www.example.com/products/index.html">Products</a>

Relative URLs

A relative URL points to a file relative to the current page. They don't include the domain name.

<a href="about.html">About Us</a> (same folder)
<a href="pages/contact.html">Contact</a> (in a subfolder)
<a href="../index.html">Home</a> (in a parent folder)
<a href="/products">Products</a> (root-relative)

Relative URLs are commonly used for internal links within the same website.

Link Types

Links can point to different types of resources:

Email Links

Create a link that opens the user's email program:

<a href="mailto:someone@example.com">Send email</a>

Telephone Links

Create a link that dials a phone number on mobile devices:

<a href="tel:+1234567890">Call us</a>

Download Links

Create a link to download a file:

<a href="files/document.pdf" download>Download PDF</a>

Creating Navigation Menus

A common use for links is to create navigation menus. Typically, navigation menus are created using unordered lists with links.

<nav>
  <ul>
    <li><a href="index.html">Home</a></li>
    <li><a href="about.html">About</a></li>
    <li><a href="services.html">Services</a></li>
    <li><a href="contact.html">Contact</a></li>
  </ul>
</nav>

The <nav> element is a semantic HTML5 tag that defines a set of navigation links.

Module 4: Images and Media

Embed images, audio, and video content into your web pages.

HTML Images

Images can improve the design and appearance of a web page. The HTML <img> tag is used to embed an image in a web page.

<img src="image.jpg" alt="Description of image">

The src attribute specifies the path to the image, and the alt attribute provides alternative text for the image.

Image Attributes

Common image attributes include:

  • src: Specifies the path to the image
  • alt: Specifies an alternate text for the image
  • width: Specifies the width of the image
  • height: Specifies the height of the image
  • title: Specifies extra information about the image
<img src="photo.jpg" alt="A beautiful landscape" width="500" height="300" title="Mountain View">

Image Formats

Common web image formats:

  • JPEG: Best for photographs
  • PNG: Best for graphics with transparency
  • GIF: Best for simple animations
  • SVG: Best for vector graphics
  • WebP: Modern format with better compression

Audio in HTML

The HTML <audio> element is used to play audio files.

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  Your browser does not support the audio element.
</audio>

Common audio attributes:

  • controls: Adds audio controls (play, pause, volume)
  • autoplay: Audio will start playing as soon as it's ready
  • loop: Audio will start over again when finished
  • muted: Audio output will be muted

Video in HTML

The HTML <video> element is used to show a video on a web page.

<video width="320" height="240" controls>
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.ogg" type="video/ogg">
  Your browser does not support the video tag.
</video>

Common video attributes:

  • controls: Adds video controls (play, pause, volume)
  • autoplay: Video will start playing as soon as it's ready
  • loop: Video will start over again when finished
  • muted: Video output will be muted
  • poster: Specifies an image to be shown while video is downloading

  4. Ensure all media elements are accessible

Module 5: Tables

Create structured data tables with headers, rows, columns, and captions.

HTML Tables

HTML tables allow you to arrange data into rows and columns. Tables are created with the <table> tag.

<table>
  <tr>
    <th>Header 1</th>
    <th>Header 2</th>
  </tr>
  <tr>
    <td>Data 1</td>
    <td>Data 2</td>
  </tr>
</table>

Table Elements

Key table elements:

  • <table>: Defines the table
  • <tr>: Defines a table row
  • <th>: Defines a table header cell
  • <td>: Defines a table data cell
  • <caption>: Defines a table caption
  • <thead>: Groups header content
  • <tbody>: Groups body content
  • <tfoot>: Groups footer content

Table Attributes

Common table attributes:

<table border="1"> - Adds border to table
<td colspan="2"> - Cell spans 2 columns
<td rowspan="2"> - Cell spans 2 rows
<th scope="col"> - Header applies to column
<th scope="row"> - Header applies to row

Advanced Table Example

<table border="1">
  <caption>Monthly Sales Report</caption>
  <thead>
    <tr>
      <th scope="col">Month</th>
      <th scope="col">Sales</th>
      <th scope="col">Profit</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>January</td>
      <td>$1000</td>
      <td>$200</td>
    </tr>
  </tbody>
</table>

Module 6: Forms and Input

Build interactive forms with various input types, labels, and validation.

HTML Forms

HTML forms are used to collect user input. The <form> element defines a form.

<form action="/submit" method="post">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name">
  <input type="submit" value="Submit">
</form>

Form Attributes

Important form attributes:

  • action: Specifies where to send form data
  • method: Specifies HTTP method (GET or POST)
  • target: Specifies where to display response
  • autocomplete: Enables/disables autocomplete
  • novalidate: Specifies form should not be validated

Input Types

Common input types:

<input type="text"> - Single-line text input
<input type="password"> - Password field
<input type="email"> - Email address field
<input type="number"> - Numerical input
<input type="date"> - Date picker
<input type="checkbox"> - Checkbox
<input type="radio"> - Radio button
<input type="file"> - File upload
<input type="submit"> - Submit button
<input type="reset"> - Reset button

Other Form Elements

Additional form elements:

<textarea> - Multi-line text input
<select> - Dropdown list
<option> - Option in dropdown
<fieldset> - Groups related elements
<legend> - Caption for fieldset
<label> - Label for input
<button> - Clickable button

Form Validation

HTML5 provides built-in form validation:

<input type="text" required> - Field is required
<input type="email"> - Must be valid email
<input type="number" min="1" max="10"> - Number range
<input type="text" pattern="[A-Za-z]{3}"> - Pattern matching
<input type="text" maxlength="10"> - Maximum length

Module 7: Semantic HTML

Use semantic elements to create meaningful, accessible web structures.

What is Semantic HTML?

Semantic HTML uses elements that clearly describe their meaning in a human- and machine-readable way. This improves accessibility, SEO, and code maintainability.

Benefits of semantic HTML:

  • Better accessibility for screen readers
  • Improved SEO (Search Engine Optimization)
  • Easier code maintenance and readability
  • Better device compatibility

Semantic Elements

HTML5 introduced several semantic elements:

<header> - Introductory content
<nav> - Navigation links
<main> - Main content
<section> - Thematic grouping
<article> - Self-contained composition
<aside> - Tangentially related content
<footer> - Closing content
<figure> - Self-contained flow content
<figcaption> - Caption for figure
<time> - Machine-readable time
<mark> - Highlighted text

Semantic Structure Example

<header>
  <h1>Website Title</h1>
  <nav>
    <ul>
      <li><a href="#home">Home</a></li>
      <li><a href="#about">About</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <h2>Article Title</h2>
    <p>Article content...</p>
  </article>

  <aside>
    <h3>Related Links</h3>
    <ul>
      <li><a href="#">Link 1</a></li>
    </ul>
  </aside>
</main>

<footer>
  <p>© 2023 Company Name</p>
</footer>

Accessibility Features

Semantic HTML works well with accessibility attributes:

<nav aria-label="Main navigation"> - Labels for screen readers
<main role="main"> - Explicit role definition
<img alt="Description"> - Alternative text
<button aria-label="Close menu"> - Button description
<input aria-required="true"> - Required field indication

Module 8: HTML5 Features

Explore advanced HTML5 elements, APIs, and modern web capabilities.

What is HTML5?

HTML5 is the latest version of HTML, introducing new elements, attributes, and behaviors that provide better structure, multimedia support, and advanced features for modern web applications.

Key improvements in HTML5 include:

  • New semantic elements
  • Native audio and video support
  • Canvas for drawing graphics
  • Local storage capabilities
  • Geolocation API
  • Improved form controls

HTML5 Semantic Elements

HTML5 introduced semantic elements that clearly define different parts of a web page:

<header> - Defines a header for a document or section
<nav> - Defines navigation links
<section> - Defines a section in a document
<article> - Defines an independent, self-contained content
<aside> - Defines content aside from the main content
<footer> - Defines a footer for a document or section
<main> - Defines the main content of a document
<figure> and <figcaption> - Defines figures with captions

Example of semantic structure:

<header>
  <h1>Website Title</h1>
  <nav>...</nav>
</header>
<main>
  <article>
    <h2>Article Title</h2>
    <p>Article content...</p>
  </article>
</main>
<footer>
  <p>Copyright information</p>
</footer>

HTML5 Multimedia Elements

HTML5 provides native support for audio and video without requiring plugins:

Video Element

<video width="320" height="240" controls>
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.ogg" type="video/ogg">
  Your browser does not support the video tag.
</video>

Audio Element

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  Your browser does not support the audio element.
</audio>

HTML5 Canvas

The <canvas> element is used to draw graphics, on the fly, via JavaScript:

<canvas id="myCanvas" width="200" height="100" style="border:1px solid #000000;">
  Your browser does not support the HTML canvas tag.
</canvas>

<script>
  var canvas = document.getElementById("myCanvas");
  var ctx = canvas.getContext("2d");
  ctx.fillStyle = "#FF0000";
  ctx.fillRect(0, 0, 150, 75);
</script>

HTML5 Form Enhancements

HTML5 introduced new input types and attributes for better form validation and user experience:

New Input Types

<input type="email"> - For email addresses
<input type="url"> - For web addresses
<input type="number"> - For numerical input
<input type="range"> - For slider controls
<input type="date"> - For date selection
<input type="color"> - For color selection
<input type="search"> - For search fields

New Form Attributes

placeholder - Hint text in input fields
required - Makes field mandatory
autofocus - Automatically focuses on the field
pattern - Specifies a regex pattern for validation
min/max - Sets minimum/maximum values for numbers
step - Specifies legal number intervals

HTML5 APIs

HTML5 introduced several JavaScript APIs for enhanced functionality:

  • Geolocation API - Gets the user's geographical position
  • Web Storage API - Stores data locally in the browser
  • Web Workers - Runs scripts in background threads
  • Drag and Drop API - Makes elements draggable
  • Server-Sent Events - Allows servers to push updates to clients
